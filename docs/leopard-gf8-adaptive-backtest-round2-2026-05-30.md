## 一、自适应逻辑回顾

```
auto 模式 (默认):
  ├─ shard_size < 64K → decomposed (零 unsafe, 小文件 cache 友好)
  └─ shard_size >= 64K → direct (大文件单遍处理最优)
```

### 4.2 分析

**R2 普遍低于 R1 (−1% ~ −7%)**:
- 原因: 系统热状态累积（连续运行 ~40 分钟基准测试）
- 所有策略同步下降，相对排序不变
- 大文件 (512K-1M) 下降最小 (−1% ~ −2%)，因为计算密集型更稳定

**变异系数 (CV) 分析**:
- 大文件 (512K-1M): CV 0.3% ~ 1.7%，非常稳定
- 中文件 (16K-256K): CV 1.6% ~ 7.9%，可接受
- 小文件 (1K-4K): CV 4.8% ~ 9.2%，噪声较大（绝对耗时短）

**R1 vs R2 的一致性**:
- 两轮中 auto 策略的相对表现一致
- 策略选择在两轮中均 100% 正确
- CV 在 R2 中并未因更多 runs 而系统性降低，说明噪声主要来自系统抖动而非采样不足
