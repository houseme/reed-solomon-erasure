## 四、观察与分析

### 4.1 吞吐量与 shard_size 的关系

- **小 shard (1K-16K)**: 吞吐量较高, SIMD overhead 占比小, LUT 表在 L1 cache
- **中 shard (64K-256K)**: 吞吐量稳定, LUT 表仍在 L1/L2 cache
- **大 shard (512K-4M)**: 吞吐量下降, LUT 表可能溢出到 L3, 内存带宽瓶颈

### 4.2 配置大小的影响

- **4x2 配置**: 吞吐量最高 (最少的 FFT 蝶形运算)
- **10x4 配置**: 吞吐量次高
- **96x48 配置**: 吞吐量较低 (蝶形运算层数多)
- **128x64 配置**: 吞吐量最低 (最多的 FFT 蝶形运算)

### 4.3 与 aarch64 的架构差异

x86_64 (AMD EPYC 9V45) vs aarch64 (Apple M5 Max):
- **小配置 (4x2, 10x4)**: x86_64 快 1.2-1.5x (AVX2 nibble-lookup 优势)
- **中配置 (32x16, 64x32)**: x86_64 快 1.3-1.8x
- **大配置 (96x48, 128x64)**: x86_64 快 2.5-3.2x (蝶形运算 SIMD 收益最大)
- **小 shard (64K)**: 某些配置 aarch64 更快 (SIMD overhead 在小数据上不利)
