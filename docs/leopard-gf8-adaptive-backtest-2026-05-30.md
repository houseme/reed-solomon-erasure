---

## 一、自适应逻辑

```
auto 模式 (默认):
  ├─ shard_size < 64K → decomposed (零 unsafe, 小文件 cache 友好)
  └─ shard_size >= 64K → direct (大文件单遍处理最优)
用户显式指定 (direct/decomposed/direct-safe) 始终优先。
```

## 六、最终结论

### 6.1 自适应策略验证: 通过

- ✅ 策略选择 100% 正确
- ✅ auto vs 最优手动策略差距 <5%（大部分 <2%）
- ✅ 10x4 配置三策略差异 <3%，auto 无劣势
- ✅ 大文件 auto = direct，性能等价
- ✅ 小文件 auto 避免 direct 策略的性能陷阱

### 6.2 推荐配置

| 场景 | 推荐 | 原因 |
|------|------|------|
| **通用** | `auto` (默认) | 自适应最优，零配置 |
| 小文件为主 | `decomposed` | 零 unsafe，小文件最优 |
| 大文件为主 | `auto` 或 `direct` | 等价性能 |
| 零 unsafe 要求 | `decomposed` 或 `direct-safe` | 完全 safe |
| 极致性能 | `direct` | 接受 1 处 unsafe |
